Presentation Outline

  • The problem

The problem

  • OME built, tested and released as a monolithic project
  • All project components are
    • either embedded in the same source code
    • or managed as submodules
  • All components are released simultaneously

The problem

  • New applications (e.g. Consortium, OMERO.figure)
  • Inability to release individual components
  • Does not scale with new applications

The problem

Example 1: Bio-Formats

  • Bug reported for a file format
  • PR opened to fix the bug
  • PR tested
    • PR merged in the development line
  • Bio-Formats release cannot happen until OMERO is ready to be released

The problem

Example 2: OMERO

  • Security vulnerability/Build problem is reported for OMERO
  • PR opened, tested internally
  • PR merged in the development line
  • OMERO is released
  • Bio-Formats version number is artificially bumped even if

The problem

Example 3: OME Consortium

  • Multiple projects have been introduced as part of the OME ecosystem
    • FLIMfit
    • OMERO.figure
    • Virtual microscopy
  • Have their own release cycle
  • Need to be integrated

General considerations

Full decoupling requires a number of elements need to be decoupled

  • Continuous Integration
    • Source code repository
    • Build system
    • CI jobs
  • Deployment
  • Documentation
  • Release

Bio-Formats decoupling

Source code repository

  • Bio-Formats has its own source code repository
  • Bio-Formats source code embedded into OMERO source code via a submodule introduced in 1d12ad1f

Bio-Formats decoupling

Build system: up to 5.1.2

  • Reference to the revision via the SHA1
  • Bio-Formats is built every time
  • Version management previously defined in Bio-Formats via ivy.xml
    ivy.xml<publications defaultconf="build,runtime">
    <artifact name="formats-gpl"/>
...
<dependencies defaultconfmapping="build,runtime->default">
    <dependency org="bf-deps" name="xalan" rev="2.7.1"/>

Bio-Formats decoupling

Build system

  • See Bio-Formats build decoupling PR
  • Submodule removed in 80925d60
  • Bio-Formats artifacts published to the OME artifactory via Maven
  • Bio-Formats version declared in omero.properties 
    omero.propertiesversions.bioformats=5.1.2
  • Bio-Formats dependencies declared in individual components 
    ivy.xml    <dependency org="ome" name="formats-bsd" rev="${versions.bioformats}" transitive="true"/>
